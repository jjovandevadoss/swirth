Analyzer Setup Guide

What IP address do I enter on the analyzer?

You enter the IP address of THIS computer (the machine running Swirth) into your analyzer's LIS/Host settings. The analyzer will connect TO this machine, not the other way around.

To find this machine's IP address:
macOS: Open System Settings → Network → select your connection → look for "IP Address"
Windows: Open Command Prompt → type ipconfig → look for "IPv4 Address"
Linux: Open terminal → type ip addr or hostname -I
Example: If this computer's IP is 192.168.1.100, you would enter:
• Host IP: 192.168.1.100 (on the analyzer)
• Port: 7000 for ASTM or 6000 for HL7 (on the analyzer)
  • 1
    On the analyzer — open LIS / Host settings. Look for a menu called LIS Setup, Host Computer, or External Output.
  • 2
    Set the protocol to ASTM E1394 (or LIS2-A2). Some analyzers label this as Serial LIS or Bidirectional ASTM. Select TCP/IP mode — not RS-232.
  • 3
    Configure the analyzer to connect to THIS machine. On the analyzer, in the Host IP or LIS Server field, enter this computer's IP address (see blue box above for how to find it). In the Port field, enter the ASTM Port shown in the Listener Settings below (default 7000).
    Example: If this computer is 192.168.1.100, enter Host=192.168.1.100 Port=7000 on the analyzer. The analyzer acts as the TCP Client; this server is the TCP Listener waiting for the connection.
  • 4
    Verify the ASTM listener is active. Check the ASTM status badge in the Listener Settings section below — it must show Listening. If it shows Not listening, restart the application.
  • 5
    Run a test transmission from the analyzer (QC sample or "Send Last Result"). Watch the Live Log Viewer at the bottom of this page for ENQ/ACK handshake and record parsing activity.
  • 6
    Create a Field Mapping profile. Go to Field Mappings, create a profile with Protocol Filter = ASTM and map result fields such as results[*].value, results[*].units, patient.id to your destination API fields. Mark it Active.
Ensure TCP port 7000 (or your configured ASTM port) is open inbound on any firewall between the analyzer and this machine.
  • 1
    On the analyzer — open HL7 / LIS settings. Look for a menu called HL7 Output, MLLP Settings, or Interface Setup.
  • 2
    Set the protocol to HL7 v2.x over MLLP (TCP). MLLP (Minimal Lower Layer Protocol) is the standard TCP framing for HL7. Configure the analyzer as the sender / client — this server is the receiver / listener. This server automatically sends an MSA|AA ACK after every message.
  • 3
    Configure the analyzer to connect to THIS machine. On the analyzer, in the Host IP or HL7 Server field, enter this computer's IP address (see blue box at top). In the Port field, enter the MLLP Port shown in Listener Settings below (default 6000).
    Example: If this computer is 192.168.1.100, enter Host=192.168.1.100 Port=6000 on the analyzer.
  • 4
    Verify the MLLP listener is active. Check the MLLP (HL7) status badge below — it must show Listening.
  • 5
    Trigger a test message. Send a test result or ORU^R01 message. The log viewer will show the MSH fields and parsed OBX observations.
  • 6
    Create a Field Mapping profile. Go to Field Mappings, create a profile with Protocol Filter = HL7 and map fields such as observations[*].value, patient.id. Mark it Active.
Ensure TCP port 6000 (or your configured MLLP port) is open inbound on this machine's firewall.
  • 1
    Configure the analyzer or middleware to HTTP POST directly to this server. Use this when the sender already speaks HTTP rather than raw TCP.
  • 2
    Use the correct endpoint for your message type: POST http://<this-machine-ip>:5001/hl7/receive (raw HL7 text body) POST http://<this-machine-ip>:5001/astm/receive (raw ASTM text body) Replace <this-machine-ip> with this computer's actual IP address (see blue box at top).
    Example: If this machine is 192.168.1.100, use:
    http://192.168.1.100:5001/hl7/receive     Send the raw message as the request body with Content-Type: text/plain.
  • 3
    For HL7 you may also POST JSON: POST /hl7/receive Content-Type: application/json { "message": "MSH|^~\\&|..." }
  • 4
    Interpret the response. 200 OK + "status":"success" = parsed and forwarded. 207 = ingested but outbound delivery failed and has been queued for retry.
The HTTP server port is controlled by the HTTP Port field in Listener Settings. Port changes require an app restart.

Listener Settings
About the "Host" fields below: These control where this server listens, NOT where it connects to. Your analyzer connects TO these ports.
• 0.0.0.0 = listen on all network interfaces (recommended — allows connections from any network)
• A specific IP (e.g., 192.168.1.100) = only accept connections on that interface
• Tip: Leave as 0.0.0.0 unless you have a specific security reason to restrict it.
